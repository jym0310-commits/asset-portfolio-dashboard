# 개발 규칙

- 모든 답변은 한국어로 작성한다.
- 코드 주석은 한국어로 작성한다.
- 변수명은 영어를 사용한다.
- 설명은 초보자가 이해하기 쉽게 작성한다.
- 변경하기 전에 무엇을 변경할지 먼저 설명한다.

앞으로

- 모든 답변은 한국어로 한다.
- 어려운 용어는 쉽게 설명한다.
- 코드를 수정하기 전에 무엇을 수정하는지 먼저 설명한다.
- 여러 파일을 수정할 경우 이유를 설명한다.
- 항상 유지보수하기 좋은 코드를 작성한다.
- 배포가 필요할 경우 아래와 같이 가이드를 줘. 
    1) 일반적인 코드 수정 (스키마 변경 없음)
        git add .
        git commit -m "모바일 반응형 레이아웃 추가"
        git push
    2) 새로운 API 키/환경변수가 필요할 때
        Codespaces의 .env 파일에만 넣으면 안 되고, Northflank에도 똑같이 등록해야 해요.
        Northflank → Services → portfolionote 1 → Environment 메뉴 → 새 변수 추가 → 저장하면 자동 재배포돼요
    3) ⚠️ DB 스키마 변경(테이블/컬럼 추가)이 필요할 때
